1.less注释
	以 // 开头的注释不会编译到css文件中
	以 /**/包裹的注释会被编译到css文件中
2.less中的变量
	使用@来声明一个变量：@color: pink;
	作为选择器和属性名：#@{selector}的形式
	作为url：@{url}
	重点：变量的延迟加载
3.less的嵌套规则
	1).基本嵌套规则
	2).&的使用 &代表的是平级关系
4.less中的混合
	混合就是将一系列属性从一个规则集引入到另一个规则集的方式
	1).普通混合									// 会编译到css文件中
	2).不带输出的混合							// 不会编译到css文件中
	3).带参数的混合
	4).带参数并且有默认值的混合
	5).带多个参数的混合
	6).命名参数									// 多个参数并且含有默认值，在给特定的参数赋值时需要加上参数名并赋值
	7).匹配模式
	8).arguments变量							// arguments表示传进来的所有参数
5.less运算
	在less中可以进行加减乘除计算
6.避免编译
	使用 ~"内容"  这种方式来实现
7.less继承 (extend)
	继承元不可以带参数，可以继承一个类也可以继承多个类
	写法: 		&:extend(.juzhong all);						// all 继承全部
	特点:		性能比混合高，但是灵活度比混合差
8.为解决问题
	选择器优先级
	&:nth-child(1) 所代表的含义
